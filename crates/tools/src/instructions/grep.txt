IMPORTANT: If the codebase-memory tool is available (via Bash), prefer it for codebase exploration — it understands code structure and returns more relevant results. Use Grep only when codebase-memory does not have the information you need, or when you need raw text/regex search rather than structural code queries.

- Fast content search tool that works with any codebase size
- Searches file contents using regular expressions
- Supports full regex syntax (eg. "log.*Error", "function\s+\w+", etc.)
- Filter files by glob pattern (eg. "*.rs", "*.{ts,tsx}") or by file type (eg. "rust", "js", "py")
- Returns file paths and line numbers with at least one match sorted by modification time
- Use this tool when you need to find files containing specific patterns
- If you need to identify/count the number of matches within files, use the Bash tool with `rg` (ripgrep) directly. Do NOT use `grep`.
- When you are doing an open-ended search that may require multiple rounds of globbing and grepping, use the Agent tool instead

Output modes:
  - "files_with_matches" (default): returns only file paths containing matches
  - "content": returns matching lines with line numbers and optional context
  - "count": returns match counts per file

Context lines (content mode only):
  - Use context_before / context_after to include surrounding lines
  - Use context to set both before and after at once

Additional notes:
  - Results are truncated at max_results (default 1000)
  - Respects .gitignore rules automatically
  - Binary files are silently skipped
